This problem is structurally similar to Diameter of Binary Tree — the path doesn't have to pass through the root, so I need to check for the best path at every node, not just once at the top. I use a helper function that returns the maximum sum achievable by extending a path through a single branch downward — that's root->val plus the larger of the left or right gain. Since node values can be negative, I clamp each child's contribution to 0 before using it, so I don't extend the path through a branch that would only hurt the sum. Separately, at every node, I check the best path that goes through both branches simultaneously — root->val plus left gain plus right gain — and update a running global maximum with that value, since that 'both branches' shape can't be returned upward, only the single-branch version can continue into the parent. I initialize the global max to INT_MIN rather than 0, because even a single negative node is a valid path. Time complexity is O(n), space is O(h) for the recursion stack.
