I'll solve this with DFS recursion rather than BFS, because the problem requires tracking the sum along a specific root-to-leaf path, and recursion naturally carries that running state down through the call stack. At each node, I subtract the node's value from the remaining target and pass that down to the children. The base case is: if I reach a leaf and the remaining target equals the leaf's value, that path is valid. I explore both left and right subtrees and return true if either one finds a valid path. Time complexity is O(n) since I visit every node once in the worst case, space complexity is O(h) for the recursion stack, where h is the tree height
